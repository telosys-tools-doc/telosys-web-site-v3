Telosys models

A Telosys model contains the entities definitions that will be used to generate the code.

Telosys uses 2 kinds of models : « Database Model » and « DSL Model »

Database model

A Database Model is created automatically by Telosys from an existing relational database.
Telosys connects to the database and discover all the tables and their relationships in order to create the model.
This kind of model is stored in XML format in a single file (the “.dbrep” file) and contains an entity for each table.

The file "databases.dbcgf" contains all the configuration for database connection.
This XML file can be edited with any editor or with one the Telosys tools.

DSL model

The DSL model type is used to create a model from scratch.
This model is composed of a set of text files (one “.entity” file for each entity).
Each entity is defined by using a very simple syntax allowing to define attributes and relationships.

For more information see the DSL syntax.

Models examples

If you want to use existing models instead of creating a model by yourself
models examples are available for download.
These models can be used to try code generation very quickly.
