Telosys models


A Telosys model contains all the entity definitions that will be used to generate the code.

DSL Model

A "Telosys model" is a model based on a set of text files written with the Telosys "Domain Specific Language".
The DSL uses a very simple grammar to define entities with their attributes and links.
Each entity is defined in a text file having the ".entity" extension (e.g. "Car" is defined in "Car.entity").



All the files defining a model can be edited with any type of text editor.
An entity is described by using a very simple syntax allowing to define attributes and relationships.

For more information see the documentation.



Note:
since Telosys 4.0 the old model in XML format (previously used for models from a relational database) no longer exists,
it is now possible to do everything with a single model format (the DSL model)

Models examples

If you want to use existing models instead of creating a model by yourself
models examples are available for download.
These models can be used to try code generation very quickly.
